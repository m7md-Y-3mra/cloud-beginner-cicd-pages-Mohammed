What is CI/CD?

Continuous Integration and Continuous Deployment (CI/CD) is a set of practices that automate the processes of building, testing, and deploying code. In cloud systems, CI/CD is essential because it allows developers to release updates rapidly and reliably without manual intervention. By automatically running checks on every code change, teams catch bugs early and ensure high software quality. Continuous Deployment then safely delivers these changes to production environments. Overall, CI/CD minimizes downtime, reduces deployment risks, and enables faster delivery of value to users in the cloud.

My Pipeline

When I push new code to the main branch, it triggers my GitHub Actions workflow. The workflow checks out my code, validates that all required files are present, and builds an artifact of my static site. Finally, a separate deployment job safely publishes this artifact to GitHub Pages, making my latest changes instantly visible online.
